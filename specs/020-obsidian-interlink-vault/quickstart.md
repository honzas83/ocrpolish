# Quickstart: Obsidian Vault Interlinking

This guide describes how to use the `interlink` command to create a connected network of documents in your Obsidian vault.

## Prerequisites
- A vault generated by `ocrpolish metadata`.
- Documents should have `archive_code`, `language`, and `references` fields in their YAML frontmatter.

## Basic Usage

To interlink all documents in a vault:

```bash
ocrpolish interlink ./my-obsidian-vault
```

## What it does

1. **Interlinks References**: Values in the "Metadata" callout's `references` field are converted to clickable links to the corresponding files.
2. **Body Interlinking**: Any mentions of archive codes in the document text are automatically linked.
3. **Language Versions**: Adds a `language_versions` row to the metadata table, linking to the same document in other languages.
4. **Smart Resolution**: If a document isn't available in the current language, it fallbacks to the English version.

## Examples

### Before Interlinking
**Metadata Callout:**
> [!info] Metadata
> | archive_code | DPC/D(69)58 |
> | language | French |
> | references | DPC/D(69)5 |

### After Interlinking
**Metadata Callout:**
> [!info] Metadata
> | archive_code | DPC/D(69)58 |
> | language | French |
> | language_versions | [English](path/to/DPC-D(69)58_eng.md) |
> | references | [DPC/D(69)5](path/to/DPC-D(69)5_eng.md) |
